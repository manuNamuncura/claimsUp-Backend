# Brainstorming Session

When a user initiates a brainstorming session with /brainstorm:

1. Begin with a Q&A format to explore the idea:
   - Ask clarifying questions about the concept
   - Focus on understanding the problem space
   - Identify key requirements and constraints
   - Explore potential solutions and approaches

2. Create structured documentation in the docs/ directory:
   - OVERVIEW.md: High-level concept and goals
   - REQUIREMENTS.md: Detailed requirements and constraints
   - PLAN.md: Implementation approach and technical details

3. Follow this process:
   - Start with open-ended questions to understand the concept
   - Guide the conversation to gather specific requirements
   - Help organize thoughts into the documentation structure
   - Ensure all key aspects are captured in the appropriate documents

4. Documentation Guidelines:
   - Use clear, concise language
   - Include relevant diagrams or visualizations when helpful
   - Maintain a logical flow of information
   - Link between documents when appropriate

5. Session Completion:
   - Review all created documents
   - Ensure all key points are captured
   - Verify document structure and completeness
   - Provide a summary of the brainstorming outcomes
