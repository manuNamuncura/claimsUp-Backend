# Feature Specification Generator

Act as a senior developer to translate a feature request into a clear, actionable technical specification.

**Usage**: Used internally by `/feature` or directly: `/spec [feature description]`

## Specification Template

### 1. Objective
- **Goal**: [A clear statement of what this feature will accomplish.]

### 2. Implementation Details
- **Inputs**: [Describe the inputs, arguments, or data the feature will receive.]
- **Outputs**: [Describe the return values or side effects.]
- **Core Logic**: [A high-level description of the steps the code will perform.]

### 3. Acceptance Criteria
- **AC-1**: [A testable condition that must be met. e.g., "Given input X, the function must return Y."]
- **AC-2**: [Another testable condition. e.g., "If the input is invalid, it must throw a specific error."]

### 4. Affected Files
- **Create**: [List of new files to be created.]
- **Modify**: [List of existing files to be modified.]
